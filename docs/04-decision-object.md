# 04. Decision Object

## Schema Reference

See [`/spec/decision.schema.json`](../spec/decision.schema.json)

## Fields

### `decision_id`
Type: `uuid`

Unique identifier for this decision instance. Generated by the Evaluator.

### `action_id`
Type: `uuid`

Identifier of the ActionEnvelope this decision applies to. Must match exactly.

### `result`
Type: `enum` — `ALLOW | DENY | HOLD`

The evaluation outcome.

- `ALLOW`: Runtime Adapter may execute the side-effect.
- `DENY`: Execution is prohibited. Action transitions to `CLOSED`.
- `HOLD`: Execution is deferred. Requires explicit re-evaluation.

### `reason_code`
Type: `string`

Machine-readable code describing the basis for the decision.
Format is implementation-defined. Must be non-empty.

Examples: `POLICY_MATCH`, `SCHEMA_INVALID`, `EVALUATOR_TIMEOUT`, `MANUAL_HOLD`

### `authority_token`
Type: `string`

Identifies the authority that produced this decision.
May reference a policy version, evaluator instance, or signing key identifier.

### `proof_hash`
Type: `string`

SHA-256 hash over `decision_id + action_id + result + timestamp`.
Enables independent verification that this decision was produced.

Required for all results including `DENY`.

### `timestamp`
Type: `date-time` (ISO 8601)

Time at which the decision was produced by the Evaluator.

## Immutability

A Decision object is immutable once produced.
The ledger must preserve the original object without modification.
